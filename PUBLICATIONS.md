## Relationship to this repository

This repo (`nexalpha_v2`) is a from-scratch rebuild of the system described in the
paper, undertaken after a code review surfaced engineering bugs not visible from the
paper's reported metrics alone — most importantly, a data-leakage bug in the backtest
module (see `README.md` → "What changed from v1").

**Practical implication**: the paper's published backtest figures (12-18% outperformance
vs. buy-and-hold, ~64% win rate) were generated by the pre-fix backtest module and should
not be assumed to reproduce on this corrected v2 codebase. The classification metrics
(test accuracy ~72-76%, AUC ~0.78) came from a standard train/test evaluation rather than
the backtest module, so they're less directly implicated, but they have not been
independently re-verified against v2 and should be treated as preliminary.

If asked about this in an interview: the honest, accurate framing is that the published
paper captures the v1 research prototype's results, and this repository documents the
subsequent engineering audit, bug discovery, and rebuild — including the discovery that
one of the paper's headline metrics rested on a leaky evaluation. Surfacing that gap is
itself part of the engineering work, not a contradiction of it.
